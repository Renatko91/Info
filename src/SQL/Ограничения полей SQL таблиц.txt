1) NOT NULL                     - не может быть null
2) CHECK
3) DEFAULT
4) UNIQUE                       - уникальное значение, допустимо множество уникальных ключей в одной таблице,
                                  может иметь null
5) PRIMARY KEY                  - первичный ключ, только один в талице,
                                  уникальный, не может быть null, может быть група полей в первичном ключе
6) FOREIGN KEY                  - внешний ключ, для связи таблиц, имеет сылочную ценность
7) Кластерный индекс            - используется для простого и быстрого извлечения данных из базы данных,
                                  он сортирует строки по столбцу, который установлен как кластеризованный
                                  индекс, только один кластеризованный индекс в таблице
8) некластеризованным индексами - тение из некластеризованного индекса происходит относительно медленнее,
                                  создает отдельный объект внутри таблицы, который указывает на исходные
                                  строки таблицы при поиске, может быть много в таблице
